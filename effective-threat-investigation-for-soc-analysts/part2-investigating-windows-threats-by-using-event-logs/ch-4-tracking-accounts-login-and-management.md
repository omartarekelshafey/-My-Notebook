### Windows Accounts

في بيئة Windows، أي عملية (Process) أو نشاط زي الـ Authentication أو الـ Object access لازم يكون مرتبط بحساب محدد. الحسابات دي هي اللي بتحدد صلاحيات التصرف داخل النظام.

#### Standard Accounts

دي الحسابات الأساسية اللي بيستخدمها الموظفين في شغلهم اليومي، ومميزاتها كالتالي:

* صلاحيات Common tasks: بتسمح للمستخدم بفتح التطبيقات، تصفح الإنترنت، والوصول للملفات المسموح له بيها.
* قيود Restricted actions: الحساب ده ملوش صلاحية تثبيت برامج أو تغيير إعدادات النظام الحساسة.
* تأمين Security impact: استخدام النوع ده بيقلل الضرر لو الحساب اتسرق، لأن المخترق مش هيقدر يوصل لملفات النظام العميقة بدون صلاحيات أعلى.

**Default Local System Accounts**

دي حسابات مخصصة للخدمات والبرامج اللي بتشتغل في الخلفية، وساعات بتشتغل قبل ما المستخدم يعمل Login أصلاً:

* حساب SYSTEM: ده أقوى حساب في الويندوز وله سيطرة كاملة (Complete control) على كل موارد النظام

### Default Local System Accounts Summary

دي حسابات مخصصة للخدمات والبرامج اللي بتشتغل في الخلفية، وساعات بتشتغل قبل ما المستخدم يعمل Login أصلاً:

*

#### 1. حساب SYSTEM&#x20;

* الوظيفة: ده حساب  ليه كامل الصلاحيات على ملفات النظام والنواة (Kernel).
* الهوية: بيتحكم في كل شيء محلياً، وبرا الجهاز بيظهر بهوية الكمبيوتر (Computer$).

#### 2. حساب NETWORK SERVICE&#x20;

* الوظيفة: مخصص للخدمات اللي محتاجة تكلم أجهزة تانية في الشبكة "بصفة رسمية".
* الهوية: جوه جهازك اسمه NETWORK SERVICE، بس أول ما يخرج على الشبكة بيستخدم  Computer Account ($) عشان السيرفرات تعرفه.

#### 3. حساب LOCAL SERVICE (المتخفي)

* الوظيفة: للخدمات البسيطة اللي ملهاش صلاحيات كبيرة ومحتاجة تتعامل مع الشبكة "بدون هوية".
* الهوية: محلياً صلاحياته ضعيفة، وعلى الشبكة بيدخل بصفة Null Session (مجهول)، مبيقدرش يستخدم بيانات الـ Computer$.

#### 4. حساب Computer Account ($)

* الوظيفة: ده "البطاقة الشخصية" للجهاز نفسه لما ينضم للـ Domain.
* الهوية: بيعرف نفسه للـ Active Directory عشان الجهاز يقدر يوصل للموارد المشتركة.

#### 5. حساب ANONYMOUS LOGON (الضيف)

* الوظيفة: بيسمح بالوصول لبعض المعلومات الأساسية من غير ما يطلب يوزر وباسورد صريحين.
* الهوية: مجهول تماماً (Guest/Anonymous Access).

### Tracking Successful Logins (Event ID 4624)

لما أي حساب (سواء كان بني آدم أو خدمة) ينجح في إنه يعمل Login، الويندوز بيسجل حدث برقم 4624. بيعرفنا مين دخل، وإزاي، ومنين.

#### Sections of Event 4624

السجل ده متقسم لـ 7 أقسام، أهمهم اللي بنستخدمه في التحقيق:

* قسم Subject: ده فيه معلومات الحساب اللي "طلب" الدخول، مش اللي دخل فعلياً. غالباً في الـ 4624 بيكون القسم ده فاضي أو ملوش لازمة أوي.
* قسم Logon Information: ده أهم حتة لأنه بيحتوي على الـ Logon Type. الرقم ده بيعرفنا المستخدم دخل إزاي (قاعد قدام الجهاز، ولا داخل ريموت، ولا خدمة...).
* قسم New Logon: ده فيه بيانات "البطل" اللي دخل فعلياً:
  * Account Name: اسم الحساب اللي دخل.
  * Account Domain: لو الاسم ده هو نفسه اسم الجهاز، يبقى ده Local Account، لو مختلف يبقى Domain Account.
  * Logon ID: رقم مميز للجلسة دي (Session) عشان تقدر تتابعها بعدين وتعرف قعد قد إيه.
* قسم Process Information: بيعرفك إيه هو البرنامج أو الـ Process اللي بدأ عملية الدخول دي، وده بيكشف لو فيه برنامج غريب هو اللي بيحاول يدخل.
* قسم Network Information: بيعرفك الجهاز اللي الشخص داخل منه (IP واسم الجهاز). ده مهم جداً عشان تكتشف الـ Lateral Movement (لو حد بيتحرك من جهاز لجهاز جوه الشبكة).

***

### Understanding Logon Types

ده الجدول اللي بيخليك "برنس" في التحقيق، لأن كل رقم ليه معنى:

| **الرقم** | **النوع (Logon Type)** | **الوصف (بالبلدي)**                                                 |
| --------- | ---------------------- | ------------------------------------------------------------------- |
| 2         | Interactive            | واحد قاعد بكرسيه قدام الجهاز وكاتب الباسورد بالكيبورد.              |
| 3         | Network                | داخل يوصل لملف شير (SMB) أو طابعة من بعيد.                          |
| 4         | Batch                  | مهمة مجدولة (Scheduled Task) هي اللي بدأت الدخول.                   |
| 5         | Service                | خدمة (Service) هي اللي عملت Login عشان تشتغل.                       |
| 7         | Unlock                 | الجهاز كان معمول له Lock والمستخدم فتحه تاني.                       |
| 10        | Remote Interactive     | داخل عن طريق RDP (سطح المكتب البعيد).                               |
| 11        | Cached Interactive     | دخل ببيانات متخزنة (لما الجهاز مش عارف يوصل للـ Domain Controller). |

#### ليه الـ Event 4624 مهم للـ SOC Analyst؟

1. بيكشف الحسابات المخترقة (Compromised Accounts).
2. بيحدد الأوقات الغريبة للدخول (زي حد داخل الفجر وهو مش ميعاد شغله).
3. بيكشف لو حد بيستخدم الـ RDP عشان يتنقل بين أجهزة الموظفين.
